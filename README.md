# Lab-7 Parallel vs Sequential I/O

## Виконані вправи
1. **SEQUENTIAL FILE READS** (`sequential_file_reads.js`) - Маршрут `GET /sequential`. Демонструє послідовне читання кількох файлів за допомогою `async/await` та вимірювання часу виконання (`elapsedMs`).
2. **PARALLEL READS** (`parallel_reads.js`) - Маршрут `GET /parallel`. Оптимізація попереднього завдання за допомогою `Promise.all(...)` для паралельного виконання незалежних I/O операцій, що суттєво зменшує загальний час виконання.
3. **HANDLE PARTIAL FAILURES** (`handle_partial_failures.js`) - Маршрут `POST /error-handling`. Використання `Promise.allSettled` для стійкої обробки масиву задач. Сервер успішно розділяє результати на масиви `successes` та `failures`, не перериваючи роботу при одиничних помилках (наприклад, при відсутності одного з файлів).
4. **THREADPOOL LIMIT** (`threadpool_limit.js`) - Маршрут `GET /threadpool-limit`. Практична симуляція впливу обмежень Threadpool у Node.js. Виконується 8 паралельних важких криптографічних операцій (`crypto.pbkdf2`), які блокують фонові потоки, з подальшим вимірюванням витраченого часу.
